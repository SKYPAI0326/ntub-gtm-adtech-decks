Step 1 · 各司其職、不要打架

三種工具
各做各的事

Option A GA4 加強型 捲動／外連／下載／站內搜尋／影片，自動追蹤六件事，能用就先用。
Option B gtag 直寫 有工程資源、邏輯簡單、單一 GA4——一行解決。
Option C GTM 容器 多平台統一管理中心，非工程師可維運，要版本控制就走它。
最常見錯誤：GA4 加強型已開、又在 GTM 加一個 click 追蹤——外連點擊被算兩次。三者可以共存，但不可雙埋同一事件。 Source · Unit 1-1 Step 1

Step 2 · 每個新需求都跑一次

四問，決定走哪條路

從上往下問，第一個「YES」就是答案。避免「什麼都丟進 GTM」。

Decision Tree · 由上而下
Q1 能交給 GA4 加強型嗎？ 能 → 停。啟用就好，不要重做。
Q2 邏輯簡單且能改碼？ 是 → gtag 一行搞定。
Q3 要接 Meta／Ads／LINE？ 要 → 非 GTM 不可。
Q4 要版本控制／非工程師維運？ 要 → 走 GTM 容器。

Compare · 兩條路的責任邊界

GA4 加強型 vs GTM 容器

Side A · GA4 加強型評估
內建六件事，自動追

捲動 ／ 外連 ／ 下載 ／ 影片 ／ 站內搜尋 ／ form interactions——勾一下就好。

適用：互動指標、單一 GA4、無 ITP 工程能量。

Side B · GTM 容器
多平台、多人、可回滾

同一個 purchase 要同步 GA4／Meta CAPI／Google Ads／LINE／TikTok——集中管理才不會走鐘。

適用：轉換事件、多媒體像素、版本紀律、跨團隊協作。

Step 1 · 三層往下走

一個 Google 帳戶，
可以管很多公司、很多站

01
Google 帳戶
同一個 Google 登入可建立多個 GTM 帳戶——通常一公司一帳戶。
02
GTM 帳戶
底下可以有多個容器。命名建議：帳戶 = 公司名。
03
容器（Container）
通常一站一容器。命名建議：容器 = 域名（如 acme.com - 主站）。
不要為「開發環境」另開一個容器——用同一個容器 + 工作區（Workspace）就好。多容器同步改動很容易走鐘。 Source · Unit 1-2 Step 1

Step 2 · 一個都不能少

Tag · Trigger ·
Variable · Folder

想像射擊的比喻：Tag 是發射器、Trigger 是發射按鈕、Variable 是彈藥。三者缺一不能射。

Folder 不影響運作，只是分類整理——建議依平台或事件類型分。

Tags
要 Fire 的東西：GA4 事件 ／ Meta Pixel ／ Ads Conversion。
發射器
Triggers
什麼時候 Fire：頁面瀏覽 ／ 點擊 ／ 自訂事件。
按鈕
Variables
Fire 時帶的動態值：URL ／ 點擊文字 ／ dataLayer 欄位。
彈藥
Folders
分類整理，建議「按平台」或「按事件類型」分群。
收納

Step 3-5 · 紀律的三件事

每改必預覽，
每預覽必驗收

Version 版本
每按「提交」＝產生快照同時發布。版本號自動遞增 v1 v2 v3，可回滾任一版。命名要寫清楚改了什麼，例：v14 - 加 TikTok Pixel + 修 purchase 重複。
Snapshot
Workspace 工作區
免費版限 3 個，邏輯像 Git branch——多人各改各的，發布時 merge。免費版限制 3 個工作區，360 版無限制。
Branch
Preview 預覽
改 → 預覽 → 連線 LocalWP/正式站 → Tag Assistant 即時顯示 Fire/Not fire。新手最常犯的錯：預覽成功就以為上線了。
Debug
Preview 模式下，你在其他分頁打開同網域也會被偵測——可以用 Incognito 測試「剛登入的新使用者流程」。 Source · Unit 1-2 Step 5

Case · 多品牌集團

三品牌、一團隊：
容器要合一還是拆三？

Before · 一容器 + hostname 條件

Tag 數量暴增到 80+，Preview Mode 卡頓；改一個品牌的 Pixel 常影響另一個。

After · 三 Web 容器 + 一 Server 容器

每品牌容器各 20 tag；Server 容器集中處理 CAPI 與 Enhanced Conversions——單品牌上線只影響自己。

隱藏坑：三容器架構下，_ga cookie 若沒設 domain=.parent.com，A 品牌跳 B 品牌會被視為兩個使用者——記得在 GA4 共享時設 cookie_domain 與 linker。 Source · Unit 1-2 實戰案例

Troubleshoot · 預覽模式報「未連線」

不要瞎猜——
跑這三個檢查點

Check 1 → 2 → 3
01 Script 有植入嗎？ DevTools Elements 搜 GTM-XXX。沒搜到 = 根本沒裝。
02 Container ID 一致嗎？ 最常見：複製舊 script 忘了換 ID。對比 GTM 後台與站上的 ID 字串。
03 廣告擋掉了嗎？ uBlock／AdGuard 會擋 googletagmanager.com。關掉再試。

三個都過了還是不行——再去查 GA4 串流是否正確（Module 1.3 第一個會踩的坑）。

Case · 3C 電商大促前

Preview Mode
搶救上線前1 小時

症狀 add_to_cart 不觸發 Tag Assistant：trigger 有 fire，但 GA4 報 Missing required parameter: items。
原因 item 前端工程師把 items 拼成 item——少了 s。
代價（如沒 Preview） −1週 大促 48 小時後才從報表發現「咦怎麼加購是 0？」流失一週再行銷受眾。
